# Estimand specification — M4.5: D-study projection to `m` raters

**Scope of this document.** The precise population definition the D-study
estimator targets: the reliability of the mean of an **arbitrary number of raters
`m`**, projected from a fitted two-way design — a generalizability-theory
**decision (D-) study**. This is exposed two ways: a downstream `d_study()`
function returning Φ(`m`) over a range of `m`, and numeric values of `icc()`'s
`unit` argument for one-off `m`. It builds directly on
[`M1-twoway-random-agreement.md`](M1-twoway-random-agreement.md) and
[`M3-incomplete-designs.md`](M3-incomplete-designs.md); the measurement model,
the variance components σ²_s / σ²_r / σ²_res, `k_eff`, and the Monte-Carlo CI are
inherited unchanged. Read those first.

This slice ships **before M5** (ROADMAP: "D-study projection"), on branch
`m4.5-d-study`. Everything numeric below is verified against closed-form oracles
and a seeded simulation — see §5.

---

## 1. What this adds to the estimand abstraction

M1 fixed the internal representation of an ICC as
`(signal component, {error component set}, averaging divisor)`. Projection moves
**only the averaging divisor**:

| Knob | prior values | this slice adds |
|---|---|---|
| Averaging divisor (**unit**) | 1 (single), `k_eff` (average) | **`m`** — any real ≥ 1 |
| Error set (**type**) | agreement / consistency | (unchanged) |
| Design interpretation (**raters**) | random / fixed | (unchanged, but see §4) |

Nothing else changes: the signal is σ²_s, the error set is `{rater, residual}`
(agreement) or `{residual}` (consistency). The divisor is now resolved by
`resolve_divisor(unit, k_eff)` — `"single" → 1`, `"average" → k_eff`,
numeric `m → m` — and stored on the estimand, so `icc_point()` needs no design
context.

---

## 2. Population definition

For a two-way **random** design the projection is the dependability coefficient
Φ at `m` raters:

```
                       σ²_s                                 σ²_s
Φ_A(m) = ──────────────────────────────    Φ_C(m) = ──────────────────────
          σ²_s + (σ²_r + σ²_res) / m                  σ²_s + σ²_res / m
```

`Φ_A` is absolute agreement (the rater main effect is in the error set), `Φ_C`
consistency. At `m = 1` these are `ICC(A,1)` / `ICC(C,1)`; at `m = k_eff` they are
the reported `ICC(A,k)` / `ICC(C,k)`. **Projection to any other `m` is an
extrapolation** to a design that was not run, deliberately named by the user and
kept out of the plain `"average"` label (M1 §5, M3 §5).

**Consistency projection is exactly the Spearman–Brown prophecy formula.** With
`ρ = ICC(C,1) = σ²_s / (σ²_s + σ²_res)`,

```
Φ_C(m) = m·ρ / (1 + (m − 1)·ρ),
```

which is the analytic oracle O-SB. (Agreement is *not* a pure Spearman–Brown of
`ICC(A,1)` because the rater term sits in both numerator-complement and
denominator at `m = 1`; its oracle O-GT is the dependability form above.)

---

## 3. Trust of the extrapolation (what the interval must show)

σ²_r is estimated from only as many raters as were observed. Projecting far beyond
that leans hard on that estimate, so the honest interval is **wide** when few
raters back it. The Monte-Carlo CI delivers this for free: it recomputes Φ(`m`) on
every draw of the components (drawn once, reused across all `m` for a coherent
curve + band), so the band widens automatically rather than plugging in a single
value. This is the reason projection uses Monte-Carlo, not the delta method
(ROADMAP; PRINCIPLES.md #3). `m` is normally an integer (a count of raters);
non-integer `m` is permitted (k_eff is itself non-integer under imbalance) and the
docs give integer guidance.

---

## 4. Fixed raters: projection is ill-posed for absolute agreement

For **fixed** raters (Case 3 / 3A) the rater term is θ²_r, the finite-population
variance of **exactly the raters observed** (M3 §6, ADR-008). There is no
"average of `m` freshly sampled raters", so projecting absolute agreement to an
`m` other than the design's own is **undefined**. This slice therefore **aborts**
(`intraclass_unidentified`, PRINCIPLES.md #5) when a numeric `unit` / `d_study()`
is requested with `raters = "fixed"` and `type = "agreement"`, pointing the user
to `raters = "random"`.

**Consistency projects freely even for fixed raters**: the rater term drops out of
the error set entirely, so Φ_C(m) is the Spearman–Brown projection of the
fixed-rater `ICC(3,1)`, which is standard. Random raters project for both types.

## 5. Oracles (PRINCIPLES.md #1 — ≥2 independent) and provenance

Verified in `tests/testthat/test-d-study.R`, regenerated by
`data-raw/oracle-d-study.R`:

- **O-SB** — consistency projection equals the Spearman–Brown formula applied to
  `ICC(C,1)` (closed form, independent of the estimator's arithmetic).
- **O-GT** — agreement projection equals the GT dependability form computed from
  the fitted components.
- **O-psych** — at `m = n_raters` the projection equals `icc()`'s own
  average-measure estimate **and** `psych::ICC`'s average-measure ICC (a third,
  independent implementation) on the balanced Shrout & Fleiss data.
- **O-sim** — a seeded simulation with known components recovers the population
  Φ(`m`) for an `m` not run (project `m = 12` from `k = 6`), and the MC interval
  covers it.
- **Sugar consistency** — `icc(unit = c(..., 3))`'s `ICC(A,3)` row equals
  `d_study(m = 3)`; projected rows carry no Shrout & Fleiss label.

Invariants also asserted: the curve is strictly increasing in `m` and stays in
`[0, 1]`; a seeded projection is reproducible and RNG-neutral (PRINCIPLES.md #9,
#12).

- **Projecting the *number of subjects*** (the other GT decision facet) — the

## 7. Multilevel rater-count projection (M17 Slice 2)

**Scope of this section.** Extends `d_study()` — until M17 it aborts on every
multilevel fit — to project the **number of raters `m`** for the multilevel
Design-1/2/3 ICCs of [`M5-multilevel.md`](M5-multilevel.md) /
[`M8-nested-multilevel.md`](M8-nested-multilevel.md). Everything in §§1–3 carries
over unchanged: projection moves **only the averaging divisor** of the *existing*
M5 estimand; the signal/error maps, `k_eff`, and the Monte-Carlo interval are
inherited. This adds no new estimand — it applies the M5 `level`-keyed estimand at
a projected divisor `m`.

**Source (#2, #4).** ten Hove et al. (2022) sanction the rater projection directly:
"different values of k can be used to estimate IRR of future … data gathered from
different numbers of raters" (p. 4), and their `k` is the **raters-per-cluster**
divisor in *both* the subject-level Eq. 12 and the cluster-level Eq. 13.

### 7.1 What is projected

For a fitted level's estimand `(signal, {error set}, divisor)`, the projected
coefficient is `signal / (signal + error / m)`:

| level | Φ(m) (agreement) | Φ(m) (consistency) |
|---|---|---|
| subject (Eq. 12) | σ²_{s:c} / (σ²_{s:c} + (σ²_r + σ²_{(s:c)r})/m) | σ²_{s:c} / (σ²_{s:c} + σ²_{(s:c)r}/m) |
| cluster (Eq. 13) | σ²_c / (σ²_c + (σ²_r + σ²_{cr})/m) | σ²_c / (σ²_c + σ²_{cr}/m) |

`d_study()` projects **each correctly-partitioned level on the fitted object**
(`subject` and/or `cluster`), producing one curve per level; the `icc_dstudy`
object gains a `level` column and `autoplot()` facets by it. Nested designs project
the **subject level only** (Design 2 two-way; Design 3 the multilevel one-way,
agreement-only), matching what `icc()` reports.

### 7.2 What is *not* projected (and why — #18)

- **Subjects per cluster.** The cluster-level ICC (Eq. 13) **contains no
  subject-related variance** — "the ordering of clusters across raters is
  independent of subject-related effects." So there is no subject divisor to move;
  the number of subjects per cluster affects only estimation **bias/efficiency**
  (the paper's Simulation 1), a sample-size/design-power matter parked in the
  ROADMAP, never a reliability curve. The earlier "three-facet / subjects-per-
  cluster" framing was dropped on this basis (ADR-026 amendment).
- **The conflated diagnostic** (`level = "conflated"`, M17 Slice 1) — a
  bias-illustration contrast, not a decision-study target; `d_study()` skips it.
- **Incomplete/ragged multilevel** — deferred: the cluster level would hit the
  open M9 `ICC(c,k)`-under-imbalance divisor (M9 §9). Complete data only for now.
- **Fixed-rater absolute agreement** — ill-posed as in §4 (no "m freshly sampled
  raters"); aborts. Fixed-rater *consistency* projects (Spearman–Brown).

### 7.3 Oracles (#1, ≥2 independent)

- **O-ML-reduction** — at `m =` the observed rater count, each level's projected
  Φ equals the fitted M5 `ICC(*,k)` for that level, to < 1e-4 (the M4.5 O-psych
  anchor, generalized per level).
- **O-ML-lme4** — the projected curve equals one built from an **independent
  `lme4::lmer`** five-component fit through the same GT form (cross-engine; the M5
  O-ML role).
- **O-ML-sim** — a seeded simulation with known components recovers the population
  Φ(m) at an `m` not run, and the Monte-Carlo band covers it (the M4.5 O-sim
  pattern).
- **Invariants** — each level's curve is monotone increasing in `m` and stays in
  [0, 1]; consistency projection equals Spearman–Brown of that level's `ICC(C,1)`.
