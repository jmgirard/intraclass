# Lessons

Durable repo lessons — build quirks, testing tricks, gotchas worth
remembering next time — captured at milestone end and surfaced at plan time.
Not status, not decisions: a lesson is a reusable "how this repo actually
behaves" note. Cross-cutting *choices* still go to `DECISIONS.md`.

Append-only; one line per lesson: `- YYYY-MM-DD (M<NN>): <lesson>`. Capped at
50 lines — when full, prune the stalest lessons rather than letting it grow;
git history keeps the full record.

- 2026-07-17 (M54): lavaan's TWO-level ML has no likelihood="wishart" (N−1) analog (unlike the single-level engine), so between-level components carry ML N-divisor shrinkage vs the REML spine; budget parity by index class (consistency near-exact, agreement asymptotic) and expect the gap to close as N_c grows, not N.
- 2026-07-17 (M55): the r-package verify slot's bare `devtools::test()` runs the live-Stan brms suite locally and cascades past the default max_fails=10 ("Maximum number of failures exceeded") on MCMC noise — NOT a regression; for a non-brms change get clean signal with `Sys.setenv(NOT_CRAN="true", CI="true"); testthat::set_max_fails(Inf); devtools::test()` (CI=true skips the skip_on_ci brms fits).
- 2026-07-17 (M56): cross-method CI-endpoint agreement (MC↔bootstrap) is a PORTABLE oracle only at the tight level (subject/consistency), NOT at the wide, few-cluster, ML-shrunk lavaan cluster level — its two-level optimizer is BLAS/OS-sensitive, so the cluster tail agreement flaked ONLY on Windows (abs .08, then rel .326 of interval width; Linux/macOS ~.02). A relative-to-width pin didn't save it. Pin cross-method CI parity where it's tight; let the wide level rest on structural sanity + the component-parity oracle (M54 glmmTMB), and always run the full CI matrix before merge — local Linux/macOS never surfaces this.
- 2026-07-17 (M59): testthat parallelism (`Config/testthat/parallel: true`) needs a `Config/testthat/start-first` listing the heaviest files, or a worker picks a slow file last and becomes the tail; helpers must be self-contained (this repo's already are). Parallel wall-clock is contention-noisy (415→205 s uncontended, but 451 s while subagents held cores) — measure per-file SERIALLY for a load-independent speed claim.
- 2026-07-17 (M59): `mc_samples` are cheap draws from the parameter vcov, NOT model refits — cutting them saves ~nothing; only `boot_samples` (each a refit) costs wall-clock. Right-size only STRUCTURAL `boot_samples` (well-formed/monotone/coherence/reproducible — B is arbitrary there, cut to the B=99 reproducibility floor + update asserted `samples` literals); every O1/O2 coverage/agreement oracle count is load-bearing for its tolerance — leave it (that's the deferred lever-b tradeoff).
- 2026-07-17 (M59): CI `air format --check .` covers `data-raw/` scripts even though `data-raw` is `.Rbuildignore`d — format new data-raw helpers or the format-check job goes red while all platform test jobs still pass.
- 2026-07-17 (M60): to freeze a bias/recovery sweep (lever b) without losing rigor, pair the frozen fixture with a same-data DIFFERENCING invariant as the live guard (SEM-minus-REML on identical data): the shared sampling noise cancels, so per-rep it sits within ~1e-3 of the deterministic target and 3-4 reps discriminate a divisor/estimator regression a frozen `.rds` can never catch (M51). Site the guard at a geometry the pilot ledger VERIFIED for differencing tightness (pilot cell B: N_c=40, k=5) — a live two-level-lavaan guard at an unverified small N_c risks false-RED under cross-version/BLAS drift ([[m56...]] Windows-only flake); keep k small so a `/(k-1)->/k` mutation shifts by ~sigma^2_r/k >> tol.
- 2026-07-17 (M60): freezing a recovery sweep can MOVE the tail, not remove it — the lavaan file dropped 137->64s but ci-bootstrap (~114s, B=99 refits) became the new `start-first` tail. A frozen file's residual cost is its remaining LIVE refits (bootstrap), so re-check which file leads after a freeze before claiming a wall-clock win.
- 2026-07-17 (M57): a fixed-rater finite-population correction makes fixed θ²_r == that engine's RANDOM σ²_r on balanced data ONLY when the random estimate is REML/unbiased (glmmTMB M37); for an engine whose random estimator is RAW (lavaan σ²_r = nu'C·nu/(k−1), τ²-inflated per ADR-014) fixed sits BELOW random by exactly bias≈τ². Don't import glmmTMB's "fixed==random on balanced" identity onto lavaan — validate fixed against glmmTMB FIXED and pin the fixed−random gap == bias as the invariant.
- 2026-07-17 (M57): to deterministically pin a fixed-rater engine's 1b-point-vs-2b-draws split without a coverage sim (extends M51), call the fit's `to_components(as.matrix(estimate))` — at the point mean the rater draw is max(0, raw − 2·bias) while the point is max(0, raw − bias), so a hand-computed bias discriminates 1b/2b/no-bias (each differs by bias). Shipping a previously-refused engine×design cell also means sweeping the engine-parity matrix (`na`→`agree`) and any abort-enumeration test, or they fail.
- 2026-07-17 (M61): a ggplot drawing overlaid series with ONE ungrouped geom_line silently sawtooths when an upstream change turns a scalar identity column into a vector (d_study's `type` went scalar→default-vector at ADR-054, zigzagging autoplot.icc_dstudy's single line). A coordinate-equality faithful-rendering test is BLIND to grouping (it passed straight through the sawtooth) — add a group-count guard (length(unique(built_line$group)) == #curves).
- 2026-07-17 (M61): when altering a plot with several latent series dimensions (type × occasions × level), render a fixture exercising EACH — the implement visual review used only non-replicate + multilevel fixtures and missed the replicate×occasions sawtooth; the diff-bug reviewer caught it. - 2026-07-21 (M75): for a BALANCED one-way random coverage sweep, the glmmTMB REML ICC(1) point equals `max(0, ANOVA-MoM rho)` exactly (σ²_a truncated at 0), so the sweep can validate coverage AND the reported point analytically — no live fits (M75 avoided ~14000 glmmTMB fits). Verify `icc()` reports exactly this point end-to-end in the fast unit suite, then run the heavy sweep fit-free against the shipped reducer in `data-raw/`. Corollary: an ICC(k) interval that is an exact monotone image of ICC(1) inherits coverage rep-by-rep — assert equality of the two coverage indicators (tolerance 0) rather than re-simulating.
- 2026-07-19 (M72): two `cairn_validate` traps for a references `Extraction:` status. (1) `references staleness` "contradicts itself" comes from a per-CLAUSE parse whose splitter merges across sentence boundaries, so a `not` in one sentence binds a later sentence's `verified`; the literal `unverified` always claims never, and `confirmed` is not a verify verb. State the status once, express exceptions without a competing verb, and locate a trip by running `_clause_claims` over the file. (2) `iso date format` is a hard FAIL that also scans WORK LOGS, so quoting a source's own non-ISO date fails the gate — paraphrase it; a disclosure quoting the offending string re-triggers the rule it discloses.
- 2026-07-21 (M77): `paths-ignore` on a `pull_request` trigger filters the PR's cumulative base…head diff, NOT the incremental push — a PR touching any non-ignored file (e.g. the workflow itself) re-runs the job on every later tracking-only sync commit; it saves work only on direct `push` events. Put it on `push` to skip docs-only commits to main; don't expect it to skip mid-PR tracking commits. To slim a matrix per-event: `config: ${{ github.event_name == 'push' && fromJSON('[...]') || fromJSON('[...]') }}` (actionlint-clean), and the PR's own run is the live self-test of which configs fire. **Correction (M78, 2026-07-21):** the "install dominates" claim is WRONG — `setup-r-dependencies@v2` already caches (install ~33–52s); CI wall-clock is the `check-r-package` **testthat suite** (ubuntu 13m/24m CPU, Windows 18m). Size CI-time estimates from the test step, not the install; the real lever is test-suite runtime (M78, D-011), not dep caching.
- 2026-07-21 (M78): testthat's parallel worker count is HARD-CAPPED at 2 by default (`testthat:::default_num_cpus()` → `getOption("Ncpus")` → `TESTTHAT_CPUS` → 2); set `TESTTHAT_CPUS=$(getconf _NPROCESSORS_ONLN)` in the R-CMD-check job to use the whole runner. But the gain is runner-bound: GitHub ubuntu is ~2.5 usable cores (2→4 workers left ubuntu testthat elapsed flat at 13m, CPU 24m→33m), only Windows benefited (18m→15m) — size CI-parallelism from ~2–3 usable cores, not getconf's logical count. Reinforces the M77 paths-ignore/cancel line: a docs-only push to a PR branch CANCELS the in-flight R-CMD-check (cancel-in-progress + the PR cumulative diff defeats paths-ignore) — hold pushes while a measurement CI run is in flight.
