## Design applicability (AC2 — read this before citing anything below)

| Axis | What the paper covers |
|---|---|
| Design | **Two-way, crossed** — "every rater scores each subject" (p. 2257, §6) |
| Raters | **Random** — "raters are randomly selected from a population of raters" (Abstract, **p. 2241**, the only place this string prints; p. 2242 restates it as raters "randomly selected from a large population of raters") |
| Interaction | **None.** Eq. (1) is additive: `Y_ij = μ + r_i + s_j + e_ij` — there is no `(rs)_ij` term, so rater×subject interaction is absorbed into `e_ij` |
| Coefficient | **Single-rating absolute agreement** — `ρ = σ²_s/(σ²_s + σ²_r + σ²_e)`, Eq. (2), p. 2242. In the package's vocabulary this is the `ICC(A,1)` family, *not* a one-way ICC and *not* consistency |
| Balance | **Balanced and complete only** — the `RS`-vector likelihood (Eqs. 16–18, p. 2258) and the Helmert-eigenvector derivation (Eqs. 30–44, pp. 2260–2261) assume every one of the `R × S` cells is observed |
| Calibration range | `ρ ∈ [0.6, 0.9]` and `δ = σ²_r/σ²_e ∈ [0.5, 16]` — see "The ρ_L = 0.6 fence" below |

**Index convention warning.** This paper uses `i = 1…R` for **raters** and
`j = 1…S` for **subjects** (Eq. 1, p. 2242) — transposed relative to
`shrout1979`/`mcgraw1996`, where `i` indexes subjects. `S` is the number of
subjects, `R` the number of raters throughout.

## Model and ANOVA layout

Eq. (1), p. 2242: `Y_ij = μ + r_i + s_j + e_ij`, `(i = 1,…,R; j = 1,…,S)`, with
`r_i`, `s_j`, `e_ij` mutually independent, mean zero, variances `σ²_r`, `σ²_s`,
`σ²_e`. Eq. (3): `σ² ≝ var(Y_ij) ≡ σ²_r + σ²_s + σ²_e`. Eq. (2) defines
`ρ = σ²_s/σ² = σ²_s/(σ²_s + σ²_r + σ²_e)`; the paper writes `ρ` and `ρ_s`
interchangeably.

**Table 1 (p. 2244)** — the ANOVA layout every method in the paper is built on:

| Source | df | SS | MS | E(MS) |
|---|---|---|---|---|
| Subjects | `S − 1` | `R Σ(ȳ_·j − ȳ_··)²` | SMS | `θ_S = σ²_e + R σ²_s` |
| Raters | `R − 1` | `S Σ(ȳ_i· − ȳ_··)²` | RMS | `θ_R = σ²_e + S σ²_r` |
| Error | `(R−1)(S−1)` | by subtraction | EMS | `θ_E = σ²_e` |
| Total | `RS − 1` | `ΣΣ(Y_ij − ȳ_··)²` | | |

Moment estimator, Eq. (4), p. 2243:
`ρ̂ = S(SMS − EMS) / [S·SMS + R·RMS + (RS − R − S)·EMS]`, attributed to
Rajaratnam (1960) and Bartko (1966); the paper notes it "is in general biased and
the resulting estimates may be negative. However, it is consistent (Fleiss and
Shrout 1979)" (p. 2243).

## The likelihood and the profile likelihood

Reparameterization, Eq. (6), p. 2245: `ρ_r ≝ σ²_r/σ²` and
`δ ≝ σ²_r/σ²_e = ρ_r/(1 − ρ_s − ρ_r)`, with `ρ_s > 0`, `ρ_r > 0`,
`ρ_s + ρ_r < 1`.

**Eq. (7), p. 2245** — the simplified `−2l`, derived in the Appendix
(Eqs. 37–40, 65):

```
−2l = c + ln[1 + (R−1)ρ_s + (S−1)ρ_r]
         + (S−1) ln[1 + (R−1)ρ_s − ρ_r]
         + (R−1) ln[1 − ρ_s + (S−1)ρ_r]
         + (R−1)(S−1) ln[1 − ρ_s − ρ_r]
         + RS ln[ (S−1)SMS/(1 − ρ_r + (R−1)ρ_s)
                + (R−1)RMS/(1 + (S−1)ρ_r − ρ_s)
                + (R−1)(S−1)EMS/(1 − ρ_s − ρ_r) ]
```

with `c` a data- and parameter-free constant (the paper prints a bare `c`, not
`c′`; Eq. 66, p. 2265, resolves it as `c₀ = (1 + ln 2π − ln RS)RS`). The paper's
own remark (p. 2245):
"the log-likelihood function is a 'function' of the ANOVA layout in Table 1" —
i.e. `(SMS, RMS, EMS)` is sufficient, which is what makes the method cheap.
The Appendix gives the four eigenvalues of the correlation matrix `V`
(Eqs. 37–40, p. 2261): `λ₁ = 1 − ρ_s − ρ_r + Rρ_s + Sρ_r`,
`λ₂ = 1 − ρ_s − ρ_r + Rρ_s`, `λ₃ = 1 − ρ_s − ρ_r + Sρ_r`,
`λ₄ = 1 − ρ_s − ρ_r`, with multiplicities `1, S−1, R−1, (R−1)(S−1)`; Eq. (64),
p. 2264 ties them back: `θ_S = λ₂σ²`, `θ_R = λ₃σ²`, `σ²_e = λ₄σ²`; Eq. (44),
p. 2261, gives `|V| = λ₁ λ₂^{S−1} λ₃^{R−1} λ₄^{(R−1)(S−1)}`, which is where the
multiplicities are read off. Eq. (65) itself prints on **p. 2264**; the notable
fact that follows it — the log-likelihood "depends only on the two parameters:
`ρ_s` and `ρ_r`" — prints on **p. 2265**.

**Profile log likelihood, Eq. (8), p. 2245:**
`l†(ρ; y) = max{ l(ρ, ρ_r; y) : 0 < ρ_r < 1 − ρ }`.

**Interval, Eq. (9), p. 2245** — the two-sided `100(1−α)%` CI is
`{ ρ : 2l†(ρ̂; y) − 2l†(ρ; y) ≤ (1 + κ) χ²_{1,1−α} }`, and **Eq. (10)** the
one-sided lower bound is the smaller root of
`2l†(ρ̂; y) − 2l†(ρ; y) = (1 + κ) χ²_{1,1−α}`. Both "are actually computed by a
one-dimensional root-finding routine nesting a one-dimensional optimization
method" (p. 2245) — the implementation shape, stated by the authors.

## The naive-PL under-coverage finding (AC4 — the load-bearing result)

`κ = 0` is the textbook profile likelihood. The paper's central negative result,
restated in every section that can carry it — Abstract, Introduction, §3.1, §4
(twice) and §6:

- Abstract, p. 2241: "the confidence intervals computed by using the profile
  likelihood function are in general too narrow to have the desired coverage
  probabilities."
- Introduction, p. 2243: "in our case, the confidence intervals of `ρ` produced
  by the traditional profile-likelihood approach are in general too narrow to
  have the desired coverage probabilities, so a remedy is necessary".
- §3.1, p. 2246: "with `κ = 0` the coverage probabilities of the resulting
  confidence intervals will be in general understated, as shown in Tables 4
  and 7."
- §4, p. 2254: "the PL approach systematically understates the coverage
  probabilities. This phenomenon is especially obvious when the number of raters
  is only three, or the number of subjects is large (`S = 50`), or the value of
  `δ` is high."
- §4, p. 2254: the confidence intervals at `κ = 0` are "unnecessarily" narrow —
  worst cell `R = 3, S = 50, δ = 4.0, ρ = 0.6`, where "the coverage rate is
  extremely low, only 79.6 %", average length 0.420 against GV's 0.604.
- §6, p. 2257: "the PL approach with `κ = 0` in general understates the coverage
  probabilities."

**Note the direction of the failure axis: it worsens as `S` grows.** More
subjects makes naive PL *worse*, not better, at fixed `R = 3` — the opposite of
the usual small-sample intuition, and directly relevant to any coverage sweep
design (cf. the M27/M28/M36 incidental-parameters lesson about sweeping the
cluster-count axis).

**Bootstrap calibration was tried and failed** (§3.1, p. 2246): 2,500 simulation
runs × 1,000 bootstrap samples each. Table 2 (p. 2246) coverage rates for a
nominal **90 %** interval range **731–862** (×1000) across nine `(ρ, δ)` cells
(the nine values, in table order: 734, 731, 757, 782, 790, 815, 844, 854, 862) —
"Clearly, the coverage rates are not satisfactory," and "the results can be even
worse." A bootstrap recalibration of PL is a recorded dead end, not an untried idea.

**Table 2 is a single-geometry table, not a sweep.** Every one of its nine cells
is `R = 3, S = 50` — the paper picks that geometry deliberately, "the case for
`R = 3, S = 50`, when the profile likelihood with `κ = 0` has the worst results"
(p. 2246). Only `ρ` and `δ` vary. So the 731–862 range is evidence about the
worst PL geometry, and the *dead end* reading above is the paper's own
generalization ("none of those bootstrap methods can serve as a rescue"), not
something Table 2 demonstrates across `(R, S)`.

## The MPL fix

`κ_corr` is the value of `κ` making coverage exact; it depends on `(ρ_s, ρ_r,
R, S)`, and the paper's verdict on computing it is flat — "A tractable expression
is hardly available for it, thus it is infeasible to use `κ_corr` in practice"
(p. 2246). It → 0 as `R` and `S`
grow. **Eq. (11), p. 2247** defines the practical substitute:

`κ_m = max{ κ_corr ≡ κ_corr(ρ, δ) : ρ_L ≤ ρ ≤ ρ_U ; δ_L ≤ δ ≤ δ_U }`

Using `κ_m` (or an estimate) in Eq. (9)/(10) **is** the modified profile
likelihood. Because `κ_m ≥ κ_corr`, MPL is deliberately **conservative**:
"the coverage probability is in general higher than the nominal level `1 − α`"
(p. 2257, §6). Grid search for `κ̂_m`: Eqs. (12)–(13), pp. 2247–2248, over
`ρ_i = ρ_L + (i−1)d` and `δ_j = r^j`; "Usually we use `d = 0.1` or `d = 0.05`
and `r = 2`, `k₁ = −1`, `k₂ = 4`" (p. 2248). The seven-step Monte-Carlo
procedure for `κ_corr` at fixed `(ρ, δ)` is spelled out on pp. 2249–2251.

### The ρ_L = 0.6 fence (the finding that most constrains reuse here)

p. 2247: "In real life, we can hardly expect a value of `ρ` that is lower than
0.6, so we can set the lower bound of `ρ` to be `ρ_L = 0.6`." Every `κ_m` the
paper tabulates is a maximum taken over `ρ ≥ 0.6`, and every simulated cell uses
`ρ ∈ {0.60, 0.75, 0.90}`. **The paper reports no evidence whatsoever at or near
`ρ = 0`** — the region this package treats as a standing known-failure axis
(GP6) and where M62/RR01 found the incumbent MC default aborts on 28–39 % of
one-way datasets. Any adoption of MPL must either recalibrate `κ_m` over a
near-zero `ρ` range or fence the method away from it; the published `κ_m` values
are not transferable there.

## Open questions

- ~~**No near-zero-ρ evidence at all.**~~ **Resolved (M87/M88).** See "The
  ρ_L = 0.6 fence" above for the original gap. M87 T2 regenerated `κ_m` over the
  extended `ρ ∈ [0.05, 0.9]` (`data-raw/m87-mpl-kappa-recalibration.R`) and M87's
  pre-registered sweep found MPL "not worse" than the incumbents across the
  near-zero boundary (GO-for-opt-in, D-014); M88 ships the extended-range `κ_m`
  as a precomputed table (`data-raw/m88-mpl-kappa-table.R` → `R/sysdata.rda`).
  The sub-0.6 `κ_m` still **carries no external oracle** (it rests on the
  package's own simulated coverage) — documented as a condition of the exported
  method (`@param ci_method`, D-014 (i)/D-015), not an unresolved obstacle.
- **The no-interaction model.** Eq. (1) omits a rater×subject interaction term,
  so `σ²_e` here is error *plus* interaction. The package's two-way random
  `ICC(A,1)` fits `subject + rater` as random effects with a residual; whether
  the two `ρ` definitions coincide term-for-term needs checking against
  `mcgraw1996` before any oracle value from Tables 4–9 is asserted against
  package output. Recorded as a mapping question, not a defect.
- **`ρ_U` is stated two ways.** §3.2, p. 2247 recommends "we can safely set
  `ρ_U = 0.98` in most cases"; the Table 3 settings (p. 2254) state `ρ_U = 0.9`.
  Table 3's `κ_m` values are therefore maxima over `ρ ∈ [0.6, 0.9]`, not
  `[0.6, 0.98]`. Use the p. 2254 figure when reproducing Table 3.
- **`κ_L`/`κ_U` are also stated twice.** p. 2254 first says "Usually, we may set
  `κ_L = −0.20`, `κ_U = 0.8` and `d_κ = 0.05`", then gives the actual Table 3
  settings as `d_κ = 0.01`, and `κ_L = 0.0, κ_U = 0.80` (two-sided, α = 0.10) /
  `κ_L = −0.20, κ_U = 1.20` (one-sided, α = 0.05). The second set is the one
  that produced the table.
- **Index-letter slip in Examples 4 and 5** (p. 2256): both write `I` for the
  rater count (`I = 3, S = 85`; `I = 4, S = 4`) where the rest of the paper
  writes `R`. Read as `R`; recorded as printed.
- **Software availability is by email.** p. 2257: the programs "are available
  from the first author upon request" plus a 2013-era personal web page
  (`www2.gsu.edu/~matyxx`). There is no package or archived code, so a live
  independent-implementation oracle is not available from the authors — an MPL
  pass would need a from-scratch implementation checked against the frozen
  tables above (the ≥2-oracle-types bar would be closed-form + frozen, or
  frozen + simulation-coverage).
- **Appendix erratum — `D` is a log-determinant, not a determinant.** The
  sentence closing Eq. (66), p. 2265, reads "`D` = `D(λ₁, λ₂, λ₃, λ₄)` is the
  determinant of the matrix `V`", but Eq. (66) defines
  `D = ln λ₁ + (S−1) ln λ₂ + (R−1) ln λ₃ + (R−1)(S−1) ln λ₄`, which by Eq. (44)
  is `ln|V|`. Harmless to the derivation (Eq. 62 uses `D` in the right place)
  and recorded only so a reimplementation does not exponentiate it.
- **Nominal level is 90 %, not 95 %,** in every two-sided table. The package's
  default is 95 %. No 95 % two-sided cell is published; the only 95 % results are
  one-sided lower bounds (Tables 7–8).
