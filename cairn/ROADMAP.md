## Candidates

- Complete the oracle registry — write entries for the oracles that ship without one, and reconcile the entries M46/M47 made stale. `ORACLES.md` states that **every** oracle value in the suite traces to an entry there, but its own header scopes it to M1–M39 and oracles kept shipping after: `O-cluster-ck` / `O-cluster-ck-cover` (M46) and `O-Bayes-cluster-ck` (M47) have no entry at all — their committed fixtures `cluster-ck-coverage-oracle.rds` and `bayesian-cluster-ck-oracle.rds` are referenced nowhere in the file — as do the `O-SEM-ML*` lavaan-multilevel family, `O-Boot-DS`, `O-IDS`, and `O-invariance`. Separately, existing entries still describe cluster `ICC(c,k)` as "open"/"dropped" on incomplete data, which M46/M47 appear to have closed; those are corrections to entries, not new ones. Sizeable enough to need its own plan (the new entries alone would blow a milestone's line cap). Promote via `/milestone-plan`. Lineage: M72 T4.
- Re-run the seeded oracle scripts to re-establish **reproducibility**, starting with `data-raw/oracle-bayesian.R`. D-008 fixed the M72 bar at *provenance* — the registry agrees with what the repo commits — and explicitly refused re-running, so no entry currently claims that a script re-run today still produces its committed values (engine, BLAS and RNG drift are all unbudgeted). M72 T4 found the concrete motivation: O-Bayes' prose disagreed with its own committed fixture on every statistic, and no re-run could adjudicate which earlier run was authoritative because a fresh run yields a third set. Scope: regenerate per script, record an engine-version delta beside each fixture, and decide what a divergence from the committed values means before running (not after). The Bayesian sweeps are multi-hour live-Stan jobs ([[brms-coverage-oracle-is-a-background-job]]), so this is background work sized per script, not one milestone for all of them. Standing gap under PRINCIPLES.md #12. Promote via `/milestone-plan`. Lineage: D-008 → M72 T4.
- Acquire the one remaining off-shelf source cited by `ORACLES.md` and verify its leg — Cronbach et al. (1972), co-cited with Brennan (2001) for the two-facet decomposition behind O-Bayes-Rep. Low stakes, and lower now that Brennan is verified: the decomposition is standard GT, is independently pinned by that entry's own oracles, and its Brennan half is confirmed against Ch. 3 (printed pp. 56/58), so what is left is one half of a co-citation. The ask is the equivalent passage in Cronbach et al. — the crossed two-facet decomposition, or better the replicates-within-cell design O-Bayes-Rep actually fits. **Narrowed from four works to one across M72 T3:** `lee2024a` and `vispoel2022b` reached the shelf mid-milestone and closed two legs (the Vispoel leg's GENOVA mis-attribution corrected in passing), and `brennan2001_ch3` closed the Brennan half. Lineage: M72 T3.
- Profile-likelihood CI pass — two-way random ICC (GO/NO-GO) — the sibling to M62, split off at the 2026-07-17 implement gate once the sources proved design-specific. Assess the **modified profile likelihood** of xiao2013 (Comput Stat 28:2241-2265; two-way random interrater, random raters) as the candidate, with naive profile-likelihood as a reference point (xiao2013 documents naive PL as under-covering). Same shape as M62: coverage-band+width criterion, GO/NO-GO, no exported method. Promote via `/milestone-plan`. Lineage: legacy candidate (`cairn/legacy/ROADMAP.md:81`) → M62 gate split.
- Unbalanced one-way transformed bootstrap-t — extend the `npbootstrap` `ci_method` (M75) to unbalanced designs: the eq. 7 IJ SE with per-subject `n_i` and the `log F` transform's `n₀`. Design work deferred at the M75 plan gate (M75 is balanced-only). **The ICC(k) Spearman-Brown pole/support alignment is balanced-only — with unbalanced data the transform's group-size constant becomes `n₀` and the MoM support changes, so the monotone-map argument must be re-derived, not assumed** (RR02 beyond-brief 2). Promote via `/milestone-plan`. Lineage: M62 GO / D-006 → M75.
- Boundary-robust classical CI for the one-way default — M62/RR01 found the glmmTMB MC default **aborts** (`intraclass_singular_fit`) on 28–39 % of near-zero-ICC one-way datasets and under-covers (0.85–0.88 conditional); a classical **SEARLE exact-F** (balanced) or **Burch REML** CI would likely dominate on normal cells and fix the *default itself* — distinct from the bootstrap-t, which does not. Assess/adopt as a one-way interval option. Lineage: M62 RR01 Q3/rec 3.
