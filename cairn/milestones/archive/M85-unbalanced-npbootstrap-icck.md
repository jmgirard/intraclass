# M85: Unbalanced one-way transformed bootstrap-t — ICC(k) via re-derived Spearman-Brown map

**Status:** done (2026-07-23, PR #92 https://github.com/jmgirard/intraclass/pull/92)

**Goal:** Extend `ci_method = "npbootstrap"` to `unit = "average"` (ICC(k)) on unbalanced one-way designs, re-deriving the Spearman-Brown pole/support alignment now that the transform's `n₀` differs from the harmonic-mean `k_eff`.

**Outcome:** `ci_method = "npbootstrap"` now serves the **unbalanced** one-way ICC(k) (`unit = "average"`, and the default `c("single","average")` call), not just the M84 ICC(1). The interval is the monotone Spearman-Brown image `g(ρ) = k_eff·ρ/(1 + (k_eff−1)ρ)` of the ICC(1) endpoints with the harmonic-mean `k_eff` divisor — the shipped `npbootstrap_ci` already realized it, so the GO branch was a **dispatch unblock** (lift the `unit="average"` abort in the `!balanced` branch of `R/icc.R`), keeping numeric `unit=m` deferred (not pole-safe: a chosen `m` may exceed `n₀`). Coverage inherits from ICC(1) as an exact event identity, unbalanced as balanced. Ships the ICC(k) leg in `ORACLES.md` O-NPBoot (AC1 pole alignment, AC4 re-derived identity, AC2 rep-by-rep inheritance over the extended n_rep=2000 sweep) plus `@param`/@details/NEWS. Reduces to M84 on `unit="single"` and to balanced M75 when equally rated.

**Decisions:** MD-1 (milestone-local) — AC1 GO: `k_eff ≤ n₀` (harmonic mean ≤ ohyama eq. 3) for **every** one-way design, proved via AM-GM on unordered triples, so the SB pole `−1/(k_eff−1)` never intrudes on the attainable ρ; numeric check `min(n₀−k_eff) = −1.4e−14` over 2×10⁵ designs. The `(RB tripwire: no-oracle)` resolved by the closed-form proof, maintainer-accepted, not escalated to Fable. Re-derived unbalanced identity `g(ρ)=k_eff(F−1)/(k_eff(F−1)+n₀)` (≠ `1−1/F` when `k_eff≠n₀`; reduces to it balanced, RR02 BC2).
